## بخش اول: درک کلی

### 🎯 این سیستم چه می‌کند؟

تصور کنید یک **دستیار تجاری ذکی** که:

- **۲۴/۷** بازارها را نظارت می‌کند
- **لحظه‌ای** فرصت‌های تجاری را شناسایی می‌کند
- **به طور خودکار** معاملات را اجرا می‌کند
- **ریسک** را کنترل می‌کند
- **گزارش‌ها** را تهیه می‌کند
- **یادگیری** از معاملات گذشته می‌کند

### 🏗️ سه لایه اصلی

```
┌─────────────────────────────────────┐
│    کاربران و داشبوردها             │  (رابط کاربری)
│  User Interface Layer               │
├─────────────────────────────────────┤
│    هسته تجارتی و هوش مصنوعی        │  (منطق کسب‌وکار)
│  Business Logic & AI Agents         │
├─────────────────────────────────────┤
│    متاتریدر ۵ و پایگاه داده       │  (ارتباط با بازار)
│  MT5 & External Services            │
└─────────────────────────────────────┘
```

## بخش دوم: اجزای اصلی

### 🔟 ۱۰ جزء استراتژیک

#### 1️⃣ **هسته متاتریدر (MT5 Core)**
**چرا نیاز است:**
- اتصال مستقیم به بازارهای حقیقی
- اجرای سفارشات فوری
- دریافت داده‌های لحظه‌ای

**مسئولیت‌ها:**
- مدیریت سفارشات (خرید/فروش)
- نظارت بر قیمت‌ها
- اجرای معاملات

---

#### 2️⃣ **موتور تحلیل قیمتی (Price Action Engine)**
**چرا نیاز است:**
- فهم الگوهای قیمتی
- شناسایی نقاط حساس (Support/Resistance)
- تشخیص روندها

**مسئولیت‌ها:**
- تجزیه‌وتحلیل نوسانات قیمت
- شناسایی الگوهای شمعی
- محاسبه سطح‌های فنی

**مثال عملی:**
```
اگر قیمت یورو/دلار از 1.0850 بالا رفت و بعد در 1.0800 توقف کرد
→ این سطح می‌تواند یک "Resistance" باشد
→ ربات یادداشت می‌کند و منتظر یک "Breakout" می‌ماند
```

---

#### 3️⃣ **موتور مفاهیم هوشمند (Smart Money Concepts Engine)**
**چرا نیاز است:**
- فهم رفتار سرمایه‌گذاران بزرگ
- شناسایی «تله‌های» بازار
- پیدا کردن فرصت‌های واقعی

**مفاهیم اصلی:**
- **Order Blocks:** جایی که موسسات خریدند یا فروختند
- **Liquidity Voids:** ناحیه‌های بدون معاملات
- **Market Structure:** شکل کلی روند

**مثال عملی:**
```
اگر بزرگ‌تاجران در 1.0900 خریدند
→ و سپس قیمت کاهش یافت
→ همین سطح می‌تواند نقطه‌ی خریدی خوبی برای ما باشد
```

---

#### 4️⃣ **موتور مفاهیم ICT (ICT Concepts Engine)**
**چرا نیاز است:**
- درک ساعات تجارتی کلیدی
- شناسایی فرصت‌های با احتمال بالا
- استفاده از ریتم‌های بازار

**مفاهیم اصلی:**
- **Fair Value Gap:** شکاف‌های قیمتی
- **BOS (Break of Structure):** شکستن سازمان
- **Kill Zones:** ساعت‌های سودآوری

**مثال عملی:**
```
در ساعت ۸ صبح توکیو (Kill Zone)
→ معمولاً بازار حرکت بیشتری دارد
→ ربات این ساعت را نظارت بیشتر می‌کند
```

---

#### 5️⃣ **سیستم هوش مصنوعی چند‌عاملی (Multi-Agent AI System)**
**چرا نیاز است:**
- تصمیم‌گیری پیچیده
- یادگیری از تاریخ
- بهبود مستمر

**عاملین:**
- **Decision Agent:** تصمیم‌گیر اصلی
- **Risk Agent:** کنترل‌کننده ریسک
- **Opportunity Agent:** شناسایی‌کننده فرصت

**مثال عملی:**
```
اگر "Opportunity Agent" یک فرصت خوب پیدا کند
→ "Risk Agent" بررسی می‌کند آیا ریسک قابل قبول است
→ "Decision Agent" تصمیم نهایی می‌گیرد
```

---

#### 6️⃣ **سیستم اخبار و هوش‌مندی (News Intelligence System)**
**چرا نیاز است:**
- درک تأثیرات اقتصادی
- جلوگیری از معاملات قبل از اخبار
- شناسایی فرصت‌های مربوط به اخبار

**مسئولیت‌ها:**
- دریافت اخبار اقتصادی
- تحلیل احساسات
- پیش‌بینی اثرات

**مثال عملی:**
```
اگر FAA (اتحادیه فدرال آمریکا) تصمیمی گرفت
→ ربات بفهمد این شامل کدام ارز است
→ معاملات را تعلیق یا فعال کند
```

---

#### 7️⃣ **موتور اسکنر بازار (Market Scanner)**
**چرا نیاز است:**
- نظارت بر ۱۰۰ + جفت ارزی
- یافتن بهترین فرصت‌ها
- پیدا کردن الگو‌های مشابه

**مسئولیت‌ها:**
- تحلیل تمام ارزها
- رتبه‌بندی فرصت‌ها
- اطلاع‌رسانی کاربر

---

#### 8️⃣ **سیستم تقویت ارز و فیلترهای تجاری (Currency Strength & Filters)**
**چرا نیاز است:**
- درک کدام ارزها قوی‌تر هستند
- فیلتر کردن معاملات ضعیف
- انتخاب بهترین فرصت‌ها

**فیلترهای اصلی:**
- **Currency Strength:** قوت ارز‌ها
- **Correlation:** ارتباط بین جفت‌ها
- **Volatility:** نوسان‌پذیری بازار

**مثال عملی:**
```
اگر دلار قوی است و یورو ضعیف
→ معامله EUR/USD (فروش) بهتر است
→ نه معامله USD/JPY
```

---

#### 9️⃣ **سیستم مدیریت ریسک (Risk Management System)**
**چرا نیاز است:**
- محافظت از سرمایه
- جلوگیری از خسارت‌های بزرگ
- کنترل استرس روانی

**قوانین اصلی:**
- **Max Loss Per Trade:** حداکثر ضرر هر معامله (۲-۳%)
- **Max Daily Loss:** حداکثر ضرر روزانه (۵%)
- **Position Sizing:** اندازه‌بندی موقعیت

**مثال عملی:**
```
اگر سرمایه ما ۱۰,۰۰۰ دلار است
→ حداکثر ریسک هر معامله = ۲۰۰ دلار
→ اگر Loss Limit برخورد کرد، تمام معاملات متوقف شود
```

---

#### 🔟 **سیستم بک‌تست و بهینه‌سازی (Backtesting & Optimization)**
**چرا نیاز است:**
- آزمایش استراتژی‌ها قبل از واقعی
- بهبود پارامترها
- پیش‌بینی کارایی آینده

**مسئولیت‌ها:**
- اجرای معاملات در تاریخ
- محاسبه نتایج
- ارائه تحلیل‌های عمیق

## بخش سوم: جریان داده

### 📊 مسیر یک معامله (Trade Journey)

```
1. بازار ↓ (داده‌های قیمتی به‌طور ثابت دریافت)
           ↓
2. اسکنر ↓ (تمام فرصت‌ها را تحلیل می‌کند)
         ↓
3. موتورهای تحلیلی ↓ (Price Action, SMC, ICT)
                   ↓
4. هوش مصنوعی ↓ (تصمیم می‌گیرد)
             ↓
5. مدیریت ریسک ↓ (تأیید می‌کند ایمن است)
              ↓
6. سفارش دهی ↓ (معامله را اجرا می‌کند)
            ↓
7. نظارت ↓ (معامله را پیگیری می‌کند)
       ↓
8. بسته‌شدن ↓ (نتیجه را ثبت می‌کند)
         ↓
9. یادگیری ↓ (از نتیجه می‌آموزد)
```

## 📚 مفاهیم اساسی برای مبتدیان

### 🎓 درس اول: استراتژی تجاری چیست؟

**تعریف ساده:**
استراتژی = مجموعه قوانینی که می‌گویند کی بخرید و کی بفروشید.

**مثال:**
```
IF قیمت از 1.0850 بالا رود
   AND در آن شمع یک "Bullish" Engulfing باشد
   AND Currency Strength مثبت باشد
THEN بخر (Buy)

ELSE اگر شرایط معکوس بود
THEN بفروش (Sell)
```

### 🎓 درس دوم: ریسک چیست؟

**تعریف ساده:**
ریسک = امکان از دست دادن پول.

**کنترل ریسک:**
```
سرمایه: 10,000 دلار
ریسک هر معامله: 2% = 200 دلار
معاملات در روز: 5
حداکثر ضرر روزانه: 5% = 500 دلار
```

### 🎓 درس سوم: سود چگونه حساب می‌شود؟

**فرمول ساده:**
```
سود = (قیمت فروش - قیمت خرید) × تعداد لات
```

**مثال:**
```
خریدم: 1.0850
فروختم: 1.0880
سود هر لات: 30 نقطه = 300 دلار
```
